# Career Program

A personal strategic management system for sustained, intentional career development over a multi-year horizon. Modeled on program management principles and adapted from Hoshin Kanri strategy deployment.

## Structure

This repository contains the three governing documents of the Career Program. Each document operates at a different level of abstraction and is updated on a different cadence.

```
career-program/
├── framework/
│   └── framework-definition.tex     # The governing constitution of the system
├── layer1/
│   └── layer1-strategic-foundation.tex     # Strategic Foundation — reviewed annually
└── layer2/
    └── layer2-operating-model.tex     # Operating Model — updated as needed
```

## Documents

**framework/framework-definition.tex**
The governing constitution. Defines the structure, purpose, and operating principles of the system. Establishes the three-layer architecture, the node hierarchy, the correction mechanism, and the methodology for populating each layer. This document changes only if the system itself is redesigned.

**layer1/layer1-strategic-foundation.tex**
The anchor document. Contains the personal mission, positioning thesis, target functions with entry angles, known gaps across competency and positioning domains, and five breakthrough goals at the five-year horizon. Reviewed and revised annually. Every element in Layer 2 traces back to something written here.

**layer2/layer2-operating-model.tex**
The living roadmap. Contains the phase structure with gate conditions, and the full node hierarchy of breakthrough goals, annual objectives, initiatives, and action items. Updated when the Layer 3 performance cadence surfaces a signal requiring recalibration.

## Layer 3

The performance cadence — weekly logs, monthly reflections, quarterly reviews, and annual reviews — is not stored in this repository. It is executed as a recurring operational rhythm and maintained in Notion as dated entries. The cadence design is defined in the framework definition document.

## Versioning

Documents are version controlled via Git. Substantive changes to Layer 1 are made only at annual reviews. Changes to Layer 2 are made whenever the cadence surfaces a signal requiring recalibration, with a commit message noting the nature of the update and what triggered it.
